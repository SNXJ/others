客户端接口

 

接口描述：用户登录

接口地址：/badou/login/sys=ios|android|web|webapp&ver=a.b.c

传输协议：HTTP POST

参数说明：{"serial":"10086","pwd":"yyyy","gps":{"lat":0,"lon":0}}

  • serial：用户名
  • pwd：密码的MD5
  • gps.lat：维度，无则传0
  • gps.lon：经度，无则传0

返回结果：

{
    "errcode":0, 
    "data":
    {
        "uid":"xxxx",
        "company":"badou",
        "access":"255",
        "tag":"badou_1030000",
        "name":"cmx",
        "desc":"",
        "hxpwd":"34a77e",
        "host":"http:\/\/mworking.cn",
        "shuffle":
        {
            "button_notice":{"name":"通知","priority":"8"},
            "button_training":{"name":"培训","priority":"7"},
            "button_exam":{"name":"考试","priority":"6"},
            "button_task":{"name":"签到","priority":"5"},
            "button_survey":{"name":"调研","priority":"4"},
            "button_chatter":{"name":"同事圈","priority":3},
            "button_ask":{"name":"问答","priority":0}
        },
        "lang":"zh",
        "admin":0
    }
}
  • uid：用户唯一标识
  • company：公司
  • access：权限码
  • tag：分组
  • name：姓名
  • hxpwd：环信登录密码
  • host：主机，客户端后续访问host/interface
  • shuffle：模块优先级及名称
  • lang：zh为中文，en为英文
  • admin：是否管理员，1是0否

客户端登录后调用极光推送的setTagsWithAlias/setAliasAndTags，用tag作为tag，用uid作为alias。

接口描述：获取资源详情 返回顶部

接口地址：/badou/viewres/sys=ios|android|web|webapp&ver=a.b.c

传输协议：HTTP POST

参数说明：{"uid":"xxxx","rid":"yyyy"}

返回结果：

{
    "errcode":0,
    "data":
    {
        "content":"\{\"e\":4\}",
        "mcnt":71,
        "ccnt":20,
        "ecnt":127,
        "eval":536,
        "url":"资源地址",
        "tag":"最新速递",
        "subject":"标题",
        "fmt":2,
        "store":1
    }
}
  • mcnt - 点赞数(废弃不用)
  • ccnt：评论数
  • ecnt：评分人数
  • eval：评分总和
  • content：用户的操作详情。json格式,e代表评分,c代表已签到。
  • url：资源内容url地址
  • tag：分类名称
  • subject：标题
  • fmt：资源格式
  • store：是否收藏，1是0否
  • task：如果是签到,带此字段
    • offline：是否过期,1是0否
    • place：签到地点
    • latitude：纬度
    • longitude：经度
    • comment：签到描述
    • deadline：截止时间
    • startline：起始时间
    • photo：是否要拍照
    • qrint：是否二维码，非0代表刷新间隔
  • entry：如果是报名，带此字段
    • offline：是否过期，1是0否
    • deadline：截止时间
    • startline：起始时间
    • deadline_c：签到截止时间
    • startline_c：签到起始时间
    • maxusr：最多报名人数
    • enroll：当前已报名人数。
    • in：0未报名，1待审核，2已通过，3已驳回
    • content：富文本html的base64编码
  • plan：如果是计划，带此字段
    • config：配置信息
    • now：当前位置，格式为"stage:offset"，比如"1:1"表示第1阶段的第1节点。如果全部学完为"done"。
